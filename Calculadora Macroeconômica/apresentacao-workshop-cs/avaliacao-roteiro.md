# Avaliação do roteiro (persona: analista sênior de Capital Strategy, cética com tempo curto)

## Persona
Analista sênior do time de Capital Strategy, lida diariamente com modelos financeiros, projeções macro e propostas a cliente. Conhece bem IPCA, SELIC, curvas de juros e expectativas Focus, mas nunca usou uma ferramenta de geração automática de apresentações e desconfia de "mágica de IA" até ver prova concreta. Quer saber, em poucos minutos, se isso resolve uma dor real do dia a dia (refazer decks de proposta) ou se é só uma demonstração bonita sem aplicação prática. Dá no máximo 15 a 20 minutos de atenção a um workshop interno antes de começar a checar e-mail.

## Notas
- Narrativa: 5/5
- Relevância: 4/5
- Takeaways: 5/5
- Evidência: 4/5

- Adequação: 4/5

## Problemas por slide
- Slide 1.2: o dado "8 a 12 slides é o padrão de uma proposta A&M" é um benchmark interno; vale deixar claro na hora da fala que vem de uma referência de estrutura de deck, não de uma pesquisa externa, para não soar como dado de mercado.
- Slide 2.1 e 2.2: são os mais técnicos do deck (nomes de módulos, fórmula de fator acumulado, códigos de série). Para esta persona isso é prova de que a ferramenta leu o código de verdade, então funciona a favor, mas o apresentador deve ter pronta uma versão de 30 segundos "em português simples" caso alguém menos técnico na sala se perca.
- Slide 2.3: o campo "Dados" ficou vazio. Não é falha grave (é o slide de recapitulação), mas se houver um número de tempo total gasto para gerar este deck, vale adicionar ali para fechar com prova concreta de agilidade.

## O que funciona
- A estrutura dor → solução → prova ao vivo → próximos passos é exatamente o que uma audiência cética de Capital Strategy precisa para sair convencida.
- Usar o próprio projeto Calculadora Macroeconômica como estudo de caso é a melhor evidência possível: a persona vê a ferramenta lendo um repositório real, não um exemplo genérico de slide de IA.
- Nenhum slide se repete em layout, e a densidade (2 blocos por slide, sem slides rasos) respeita o tempo curto que a persona dá ao conteúdo.
- O fechamento com link e QR code para a ferramenta é o gancho certo para essa persona "cética mas pragmática": ela só age se tiver o caminho imediato para testar.
